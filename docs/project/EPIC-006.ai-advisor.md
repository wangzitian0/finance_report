## 🎯 目标

基于 Gemini 3 Flash 构建对话式 AI 财务顾问，帮助用户理解财务状况、解读报表、回答财务问题。

**核心原则**:
```
AI 只做解读和建议，不直接修改账本
数据仅本地处理，不上传第三方
明确标注"仅供参考"
```

## 📚 Prompt 设计

### System Prompt

```
你是一位专业的个人财务顾问。你的职责是：
1. 解读用户的财务报表和数据
2. 回答财务相关问题
3. 提供专业但易懂的建议

你必须遵守以下规则：
- 只能读取用户的财务数据，不能修改任何内容
- 回答必须基于真实数据，不能虚构
- 每次回复末尾添加："以上分析仅供参考。"
- 如果用户询问非财务问题，礼貌告知这超出你的能力范围
- 使用用户的语言回复（中文或英文）

用户财务概况：
- 总资产: {total_assets}
- 总负债: {total_liabilities}
- 净资产: {equity}
- 本月收入: {monthly_income}
- 本月支出: {monthly_expense}
- 未匹配交易: {unmatched_count} 笔
```

### 典型对话

```
用户: 我这个月的支出为什么这么高？
AI: 您本月支出 5,200 SGD，较上月增加 30%。主要原因是：
1. 餐饮支出 1,800 SGD（+800 较上月）
2. 购物支出 1,200 SGD（+400 较上月）
3. 交通支出 500 SGD（持平）

建议关注餐饮支出的增长，可以考虑设置月度预算限额。

以上分析仅供参考。
```

## ❓ Q&A (待确认问题)

### Q1: AI 服务可用性要求
> **问题**: 如果 Gemini API 不可用，如何处理？

**✅ 你的回答**: A - 显示错误提示，等待恢复

**决策**: 优雅的错误处理，无降级方案
- 当 OpenRouter 返回配额不足或超时时：
  - 捕获异常：`OpenRouterQuotaExceeded`, `APITimeout` 等
  - 返回用户友好的错误提示：
    ```json
    {
      "error": "AI 服务暂时不可用，请稍后重试",
      "message": "今日额度已用完，明天再来聊吧！"
    }
    ```
  - 前端显示：聊天框禁用，显示重试按钮和重试时间估计
  
- **监控与告警**:
  - 记录所有 API 失败到日志
  - 关键错误发送告警通知
  
- **恢复机制**:
  - 定时健康检查（每 5 分钟一次）
  - 恢复后自动重新启用聊天功能

### Q2: 会话历史保留时长
> **问题**: 用户的聊天记录保留多久？

**✅ 你的回答**: C - 永久保留（用户可手动删除）

**决策**: 完整的会话历史管理
- **数据模型**:
  ```
  ChatSession:
    id, user_id, created_at, title (自动生成或用户设置)
  
  ChatMessage:
    id, session_id, role ('user'/'assistant'),
    content, created_at, metadata (tokens, model_used, etc.)
  ```
- **存储策略**:
  - 所有聊天记录永久保存到数据库
  - 用户可查看历史会话列表
  - 支持按日期、关键词搜索历史
  
- **删除管理**:
  - 用户可删除单条消息（标记为 deleted，不真正删除）
  - 用户可删除整个会话
  - 支持批量删除
  - 删除后不可恢复（UI 确认对话）
  
- **隐私**:
  - 聊天内容仅存储用户私有数据库
  - OpenRouter API 调用时，不持久化敏感信息到第三方
  - GDPR 合规：支持数据导出和完全删除

### Q3: 免责声明的形式
> **问题**: 免责声明如何呈现？

**✅ 你的回答**: C - 首次使用时弹窗确认

**决策**: 一次性同意 + 持续提示
- **首次进入聊天页面时**:
  - 显示模态弹窗，包含完整免责声明
  - 用户必须勾选 "我已阅读并同意" 才能开始聊天
  - 记录用户同意时间和版本号（如需更新条款）
  
- **免责声明内容**:
  ```
  ⚠️ 免责声明
  
  本 AI 财务顾问的回复基于您提供的财务数据生成，
  但可能包含错误或遗漏。
  
  所有分析和建议仅供参考，不构成专业财务建议。
  
  在做任何重要财务决策前，请咨询持证财务顾问。
  
  我们不对因使用本工具而导致的任何损失负责。
  ```
  
- **持续提示**:
  - 每条 AI 回复末尾显示小提示：
    "💡 此分析仅供参考，不构成投资建议"
  - 页面底部固定脚注链接到完整条款
  
- **用户管理**:
  - 用户可在设置中重新阅读免责声明
  - 如条款更新，需要用户重新同意

### Q4: API 调用限制
> **问题**: 如何限制 AI 调用以控制成本？

**✅ 你的回答**: A - 无限制（依赖 OpenRouter 层面的限流）

**决策**: 应用层无需限制，依赖 OpenRouter
- 成本控制已在 OpenRouter 层面：每天 $2 配额
- 应用层无需实现额外的调用限制
- 当 OpenRouter 返回配额耗尽时，按 Q1 的方案处理（显示错误）
- 可选的使用统计（不作为限制）：
  - 记录每个用户的月度调用次数
  - 在用户面板显示"本月已使用 X 条消息"
  - 仅供信息展示，不作为强制限制

### Q5: AI 能否主动提醒
> **问题**: AI 是否应该主动推送提醒？

**✅ 你的回答**: A - 仅被动回答问题，不主动推送

**决策**: AI 严格被动模式
- AI 财务顾问仅在用户主动提问时响应
- 不生成主动推送、提醒、或通知
- 不在仪表板显示 AI 洞察卡片
- 好处：
  - ✅ 简化实现（不需要后台任务）
  - ✅ 用户可完全控制交互时机
  - ✅ 避免 AI 推送导致的决策偏差
  
- **后续可能的扩展** (v2.0+):
  - 用户可在设置中选择启用"每周财务摘要"（但不推荐）
  - 仅生成统计摘要，不涉及 AI 建议
